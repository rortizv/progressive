This page uses RenderMode.Prerender: Angular rendered it exactly once during npm run build, and the timestamp above is frozen from that moment. Reload this page as many times as you want — it never changes, because the server is handing back the exact same static HTML every time.

Compare that to the home page, which uses RenderMode.Server — a fresh render (and a fresh call to /api/health) on every single request.

Both render modes are declared per-route in app.routes.server.ts, so a real app can mix static marketing pages with dynamic, data-driven ones freely.
